# Key Features Overview

## 1.2.1 7 Core Features

1. **Stream-based Design**: All message outputs are asynchronous stream-based designs supporting real-time responses
2. **Cookie Security Detection**: Built-in cookie security detection functionality to protect session security
3. **Provider Independent Mechanism**: Data types and conversation management are independent of specific vendors, abstracted as 'model adapter' interface, offering high portability
4. **MCP Client Support**: Extension mechanisms integrate MCP clients, enhancing system expansion capabilities
5. **Event-Driven Architecture**: Comprehensive event system with runtime dependency injection mechanism for flexible and reactive agent behavior
6. **Tool Integration Framework**: Robust system for integrating external tools and services
7. **Advanced Memory Management**: Sophisticated context handling with automatic context compression and token optimization

## 1.2.2 Architectural Advantages

AmritaCore's architecture is designed with modularity in mind. The core components can be used independently or together, allowing for:

- Easy customization to specific use cases
- Incremental adoption of features
- Reduced coupling between components
- Simplified testing and maintenance

## 1.2.3 Application Scenario Examples

**Scenario 1: Customer Support Bot**
A bot that can access knowledge bases, process orders, and escalate complex issues to human agents.

**Scenario 2: Research Assistant**
An agent that can gather information from multiple sources, analyze documents, and synthesize reports.

**Scenario 3: Workflow Automation**
An intelligent system that orchestrates various tools and services to automate business processes.

These scenarios benefit from AmritaCore's event-driven architecture, tool integration capabilities, and robust memory management.
